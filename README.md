## Обзор проекта

Дашборд пользователей - это SPA-приложение для просмотра, поиска, фильтрации и управления пользователями. Данные загружаются с [DummyJSON API](https://dummyjson.com/docs/users). 

### Основные возможности

- Таблица пользователей с сортировкой
- Поиск с оптимизацией
- Фильтрация по роли и полу
- Пагинация с настраиваемым количеством страниц
- Детальная карточка пользователя
- Создание новых пользователей
- Удаление пользователей с подтверждением
- Поддержка тёмной темы
- Полная accessibility (ARIA-атрибуты)

## Технологический стек

| Технология | Версия | Назначение |
|-----------|--------|------------|
| **Next.js** | 16 | Фреймворк |
| **TypeScript** | 5.x | Строгая типизация |
| **Tailwind CSS** | 4.x | CSS-фреймворк для стилизации |
| **TanStack Query** | 5.x | Управление серверным состоянием и кэшированием |
| **Axios** | 1.x | HTTP-клиент с перехватчиками |

### Почему именно этот стек?

#### Next.js 16 
- **Серверные компоненты** по умолчанию уменьшают размер клиентского бандла
- **Файловая маршрутизация** упрощает навигацию
- **Встроенная оптимизация** изображений и шрифтов
- **API Routes** позволяют при необходимости добавить бэкенд

#### TypeScript
- **Безопасность типов** предотвращает целый класс ошибок на этапе компиляции
- **Автодополнение** в IDE ускоряет разработку
- **Самодокументирующийся код** через интерфейсы и типы

#### Tailwind CSS + методология БЭМ
- **Tailwind** обеспечивает консистентный дизайн через utility-классы
- **БЭМ** обеспечивает понятность структуры и самодокументацию
- **@apply** директива комбинирует преимущества обоих подходов

#### TanStack Query (React Query)
- **Автоматическое кэширование** снижает количество запросов
- **Инвалидация кэша** после мутаций поддерживает данные актуальными
- **Оптимистичные обновления** (потенциально) улучшают UX
- **Повторные запросы** при ошибках повышают надёжность

#### Axios
- **Интерсепторы** для централизованной обработки ошибок
- **Автоматический парсинг JSON**
- **Таймауты** из коробки
- **Отмена запросов** через AbortController

## Архитектура проекта

Проект построен по методологии **Feature-Sliced Design (FSD)**:
- 🎯 **Разделение ответственности** между слоями
- 🔒 **Изоляцию** бизнес-логики от UI
- ♻️ **Переиспользование** компонентов
- 📈 **Масштабируемость** при росте проекта

## Процесс разработки

Разработку проекта начал с проектирования, включавшего анализ API эндпоинтов DummyJSON, создание TypeScript-интерфейсов на основе ответов сервера и декомпозицию интерфейса на независимые компоненты; в качестве архитектурного решения выбрал методологию FSD для обеспечения масштабируемости. Далее перешел к настройке инфраструктуры: инициализация Next.js с TypeScript и Tailwind, конфигурация Axios с базовым URL и интерцепторами, внедрил TanStack Query с глобальными настройками кэширования и организовал файловую структуру по слоям FSD. На уровне базовых переиспользуемых компонентов создал значки для статусов и ролей, спиннер для индикации загрузки, EmptyState и ErrorState компоненты с кнопкой повтора для обработки соответствующих состояний. Реализовал слой сущностей, включивший полный интерфейс User с вложенными объектами, API-функции с объектными параметрами, константы ролей и цветовых схем, а также хук с поддержкой поиска, фильтрации и пагинации.

На уровне фич реализовал независимые пользовательские сценарии: поиск с debounce-логикой на 300 мс, кнопкой очистки и поддержкой клавиши Escape; фильтрацию с выпадающими списками, счётчиком активных фильтров и автоматическим сбросом пагинации при изменениях; пагинацию с динамическим расчётом видимых страниц и двумя режимами отображения; удаление с модальным окном подтверждения, мутацией через React Query и автоинвалидацией кэша; создание пользователя через многосекционную форму с валидацией и трансформацией данных. Эти фичи скомпоновал: таблицу пользователей с сортируемыми колонками, цветовыми бейджами ролей и поддержкой клавиатурной навигации, а также модальное окно с детальной информацией, маскированием номера карты и моноширинным шрифтом для крипто-кошельков.

На финальном этапе всё собрал на странице дашборда. Для стилизации применил Tailwind для базовых стилей через директиву @apply и БЭМ-нейминг, а также тёмную тему с использованием Tailwind-классов dark:

## Функциональность

### Таблица пользователей
- Сортировка по имени и возрасту
- Визуальные индикаторы сортировки
- Аватары с hover-эффектом
- Цветовые бейджи для ролей

### Поиск и фильтрация
- Мгновенный поиск с debounce
- Фильтрация по роли (admin, moderator, user)
- Фильтрация по полу
- Счётчик активных фильтров

### Детальная карточка пользователя
- Вся информация о пользователе
- Секционная структура
- Маскирование конфиденциальных данных

### Создание пользователя
- Многосекционная форма
- Валидация полей
- Автоматическое закрытие после успеха

### Удаление пользователя
- Модальное окно подтверждения
- Индикатор загрузки
- Автообновление списка

### Оптимизации
- Debounce на поисковые запросы
- Кэширование через React Query (5 минут stale time)
- Подставные данные для плавных переходов между страницами
- Мемоизация вычислений (useMemo, useCallback)

Проект готов к расширению: добавлению новых фич (редактирование, массовые операции), интеграции с реальным бэкендом, добавлению тестов и Storybook.
